## 🧪 Тестирование экрана Explore (Поиск)

### Чек-лист тестирования

#### ✅ Загрузка данных

- [ ] При открытии экрана показывается ActivityIndicator
- [ ] Данные загружаются за разумное время (< 3 сек)
- [ ] Список заполняется объектами
- [ ] Каждая карточка содержит все требуемые поля

#### ✅ Карточки объектов

- [ ] Изображение отображается (или placeholder если URL неверный)
- [ ] Название видно полностью или обрезано (…) на 2 строки
- [ ] Город и тип показываются рядом
- [ ] Описание обрезано на 2 строки
- [ ] Цена форматируется с пробелами: "15 000 ₸"
- [ ] Цена показывает "/ ночь" текст

#### ✅ FlatList функциональность

- [ ] Скролл плавный и не дёргается
- [ ] Карточки не исчезают при скролле
- [ ] Пусто список отображает пустое состояние
- [ ] При много элементах виртуализация работает (рендер только видимых)

#### ✅ Навигация на детали

- [ ] Нажатие на карточку переводит на `property/[id]`
- [ ] ID в URL правильный
- [ ] Можно вернуться назад (back button)

#### ✅ Фильтры - открытие/закрытие

- [ ] Кнопка "Фильтры" нажимается
- [ ] Модаль открывается с анимацией
- [ ] Кнопка X закрывает модаль
- [ ] Нажатие на фон закрывает модаль

#### ✅ Фильтры - город

- [ ] TextInput вводит текст
- [ ] Клавиатура появляется и исчезает правильно
- [ ] Текст сохраняется при переключении между полями

#### ✅ Фильтры - тип жилья

- [ ] Все 4 кнопки отображаются (Все типы, Квартира, Дом, Комната)
- [ ] Выбранная кнопка подсвечивается синим
- [ ] Можно переключаться между типами
- [ ] "Все типы" снимает выделение с других

#### ✅ Фильтры - цена

- [ ] Оба поля введа текст
- [ ] Работает только числовой ввод (keyboard type = numeric)
- [ ] Минимальная цена может быть пустой
- [ ] Максимальная цена может быть пустой

#### ✅ Кнопки действий

- [ ] "Применить" закрывает модаль и отправляет запрос
- [ ] "Сбросить" очищает все поля и загружает все объекты
- [ ] Оба действия работают быстро

#### ✅ Обработка ошибок

- [ ] При ошибке загрузки показывается сообщение об ошибке
- [ ] Кнопка "Попробовать снова" переновных попытку
- [ ] После успеха ошибка исчезает
- [ ] При неправильном IP адресе показывается ошибка сети

#### ✅ Badge с количеством фильтров

- [ ] При 0 активных фильтров - badge не показывается
- [ ] При 1+ активных фильтров - badge показывает число
- [ ] Число обновляется при изменении фильтров

#### ✅ Пусто состояние

- [ ] Если объектов 0, показывается сообщение "Объектов не найдено"
- [ ] Иконка Search отображается
- [ ] Текст предлагает изменить фильтры

### Примеры тестовых данных

#### Тест 1: Загрузка всех объектов
```
Действие: Открыть экран Explore
Ожидаемо: Список заполнен объектами
Проверка: properties.length > 0
```

#### Тест 2: Фильтр по городу
```
Действие: 
  1. Открыть фильтры
  2. Ввести "Алматы"
  3. Нажать "Применить"
Ожидаемо: API вызов: /properties?city=Алматы
Результат: Только объекты из Алматы
```

#### Тест 3: Фильтр по типу
```
Действие:
  1. Открыть фильтры
  2. Выбрать "Дом"
  3. Нажать "Применить"
Ожидаемо: API вызов: /properties?type=дом
Результат: Только дома
```

#### Тест 4: Фильтр по цене
```
Действие:
  1. Открыть фильтры
  2. Ввести: От 10000, До 50000
  3. Нажать "Применить"
Ожидаемо: API вызов: /properties?minPrice=10000&maxPrice=50000
Результат: Объекты в диапазоне 10k-50k
```

#### Тест 5: Комбинированный фильтр
```
Действие:
  1. Открыть фильтры
  2. Город: Алматы
  3. Тип: Квартира
  4. От 20000 До 100000
  5. Нажать "Применить"
Ожидаемо: 
  GET /properties?city=Алматы&type=квартира&minPrice=20000&maxPrice=100000
Результат: Квартиры в Алматы 20k-100k
```

#### Тест 6: Сброс фильтров
```
Действие:
  1. Применить какие-нибудь фильтры
  2. Открыть фильтры
  3. Нажать "Сбросить"
Ожидаемо: 
  - Все поля очищены
  - GET /properties без параметров
  - Показаны все объекты
```

#### Тест 7: Навигация
```
Действие: Нажать на карточку с ID=5
Ожидаемо: Перейти на /property/5
Результат: Экран деталей открыт
```

#### Тест 8: Ошибка сети
```
Действие: 
  1. Отключить интернет (WiFi/Bluetooth)
  2. Открыть экран / попробовать загрузить
Ожидаемо: Ошибка "Ошибка при загрузке объектов"
Результат: Кнопка "Попробовать снова" видна
```

### Тестирование производительности

#### FlatList оптимизация
```
Проверка: 
  1. Загрузить 100+ объектов
  2. Скролить быстро вверх-вниз
Ожидаемо: 
  - Плавный скролл без фриза
  - FPS > 50
  - Нет lag/stutter
```

#### Размер карточек
```
Проверка:
  1. Каждая карточка загружает изображение
  2. Множество карточек на экране
Ожидаемо:
  - Быстрая загрузка (< 100ms на карточку)
  - Изображение не растягивается
  - Размер правильный (200x height)
```
